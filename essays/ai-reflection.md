## Introduction

The integration of Artificial Intelligence into education has shifted from a futuristic concept to a daily reality, especially within Software Engineering. In a field defined by rapid syntax evolution and complex architectural patterns, AI acts as both a high-speed calculator and a tireless tutor. Throughout ICS 314, I primarily utilized Claude for structural analysis of my essays. While these tools offer an incredible "head start," the course challenged me to balance their efficiency with the need for genuine cognitive struggle.

## Personal Experience with AI

Experience WODs: I used AI sparingly here. It was helpful for interpreting error logs, but I avoided asking it to write the code because the 30-minute timer makes it easy to "cheat" yourself out of learning the muscle memory required for the exam.
In-class Practice WODs: Similar to with the experiences, I used AI to debug my code. Primarily, I would use it to interpret error logs and then find a soltuion myself.
In-class WODs: For the in class WODs, I would often use AI to generate an initial solution and then debug it/ensure it meets the requirements myself. This approach was especially helpful when our In class WODs would build off of a previous WOD or practice WOD, such as when we were learning Bootstrap.
Essays: I used AI for outlining. Prompt: "Given these three themes from my technical experiences, suggest a logical flow for a 500-word blog post." It helped overcome "blank page syndrome," though I wrote the actual prose myself to keep the voice authentic.
Final Project: For Study-Viser, I used Gemini to assist me in implementing the course map. Specifically, I used it to generate a table of building names to coordinates that I used in my code to display the location of courses given only the building and room name.
Learning a concept / tutorial: Extremely useful. Prompt: "Explain the difference between 'export default' and 'export' in ES6 like I'm a beginner." The benefit was an instant, tailored explanation that Google searches often obscure.
Answering a question in class or in Discord: I didn't use AI here. I felt that if I couldn't answer a peer’s question from my own knowledge, I shouldn't be answering it at all.
Asking or answering a smart-question: Before posting to Discord, I’d ask ChatGPT: "Does this question include enough context for a developer to help me: ?" It helped me realize when I was being too vague.
Coding example (reduce function): I asked ChatGPT to show me an example of how to use the JavaScript reduce function and then I used that example and tweaked it to match my use case. It was faster than digging through the reduce documentation.
Explaining code: I did not use AI to explain how/why code works the way it does. This is because I feel that although AI is useful for building, it is the developer's job to understand it. For me personally, the way software works sticks better in my head if I am able to figure out the way it works myself rather than have AI explain it to me.
Writing code: I used Claude for repetitive tasks. Prompting it to "Write a mapping function for this array" saved me minutes of typing, but I always had to double-check the logic.
Documenting code: This was a huge time-saver. Prompt: "Generate JSDoc comments for this function." It’s a tedious task that AI handles perfectly.
Quality assurance (e.g., Fix ESLint errors): I used the prompt: "Why is ESLint complaining about 'no-unused-vars' here when I'm using it below?" It helped me understand scoping issues I had overlooked.
Other uses (Portfolio management): I used AI to help debug my Jekyll configuration for my portfolio site when my CSS wasn't loading correctly on GitHub Pages.

## Impact on Learning and Understanding

AI has been a double-edged sword. On one hand, it has drastically lowered the barrier to entry for complex frameworks like React and NextJS. My comprehension of Functional Programming improved because I could see multiple ways to refactor the same code instantly. However, the challenge is "learned helplessness." There were moments where I found myself reaching for a prompt before even trying to think through a bug, which can stunt the development of deep problem-solving intuition.

## Practical Applications

Outside of class, I applied these AI workflows during the HACC (Hawaii Annual Code Challenge). We used AI to quickly generate mock data for our database, which allowed us to build a working prototype in half the time. It proved that in real-world scenarios, AI is an incredible force-multiplier for "plumbing" tasks, leaving more room for creative architecture.

## Challenges and Opportunities

The biggest challenge is hallucination. Sometimes, you spend more time debugging code that AI writes rather than if you just wrote it yourself. The opportunity here is for the curriculum to include "AI Debugging" as a formal skill: learning how to verify AI output against official documentation.

## Comparative Analysis

Traditional methods (reading textbooks, watching lectures) are superior for building a foundational "mental map." However, AI-enhanced learning excels in applied skill development. While a lecture explains what a Join is in SQL, AI can show me five ways to write it for my specific project in seconds. Retention is higher when the information is applied immediately to a problem I’m currently facing.

## Future Considerations

In the future, I expect AI to be integrated directly into the IDE as a "Pair Programmer" that understands the entire codebase, not just snippets. The challenge for ICS 314 will be ensuring students don't skip the "struggle phase" of learning. We might see more exams moving toward "Explaining the code" rather than just "Writing the code."

## Conclusion

My experience in ICS 314 has shown that AI is an indispensable tool for the modern developer, but it is not a replacement for fundamental knowledge. It is a powerful bicycle for the mind, but you still have to know how to pedal. For future courses, I recommend leaning into AI as a tool for code review and documentation, while keeping the core logic assessments strictly "human-only" to ensure the foundation remains solid.
